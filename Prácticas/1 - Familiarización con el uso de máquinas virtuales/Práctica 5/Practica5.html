Práctica 5: Diferencias entre redes host-only, NAT y bridged

1. Objetivo

A la hora de crear las conexiones de red virtuales, todos los hipervisores nos van a permtir una de tres opciones para la conectividad de la tarjeta de red virtual:



  1. Host-only: El NIC (Network Interface Controller) virtual está conectado a un conmutador virtual (VMnet1) que sólamente permite intecambiar tráfico de red con el mismo hipervisor, o con otras máquinas virtuales, dentro de ese hipervisor, que estén conectadas a esa misma red host-only.

  2. NAT: El NIC virtual está conectado a un conmutador virtual (VMnet8) a través del cual la máquina virtual puede acceder, usando la tarjeta de red física del ordenador, a la misma red real a la que tiene acceso el host. La máquina virtual tendrá su propia dirección IP, distinta de la del host (y habitualmente proporcionada por DHCP por el propio hipervisor), pero a la hora de acceder al exterior el hipervisor usa NAT (Network Address Translation, aunque para ser más exactos la técnica usada es PAT = Port Address Translation) para modificar las cabeceras de los paquetes de red, de forma que la dirección IP con la que la máquina virtual es vista desde la red externa es la misma que la usada por el host. Por supuesto, la máquina virtual se puede comunicar nativamente (sin necesidad de hacer NAT) con cualquier otra máquina virtual que tenga un NIC virtual conectado a la red NAT, VMnet8.

  3. Bridged: En este modo el NIC virtual está conectado a un comnutador virtual (VMnet0) a través del cual tiene acceso directo e indiscriminado a la tarjeta de red del host, de forma que la máquina virtual podrá acceder a la red real a la que tiene acceso el host, pero usando directamente sus propias direcciones IP y MAC, que serán distintas de la del host. Obviamente, en este modo la máquina virtual deberá tener asignada una dirección IP válida, que no entre en conflicto con el direccionamiento y las asignaciones IP ya existentes en la red real. Adicionalmente, la máquina virtual estará totalmente expuesta a accesos desde la red real, sin protección ninguna del host, por lo que es mandatorio que tenga configurados mecanismos de seguridad, especialmente un cortafuegos.
En esta práctica usaremos una máquina XP, creada con un clon enlazado, e iremos modificando la configuración de su NIC virtual para comprobar las diferencias en el comportamiento de las distintas opciones de red virtual.
