Ejercicio 1  - Instalación de TomatoCart sobre un único servidor


En este Ejercicio instalaremos el software de comercio electrónico Tomatocart en una única máquina virtual, que albergará, pues, el servidor web Apache, la lógica de negocio PHP de Tomatocart, y la base de datos MySQL. Una vez instalado el software procederemos también a usar su interfaz administrativo para configurar la tienda virtual ofrecida.

Durante la instalación se crearán también varias instantáneas; en particular, estas instantáneas nos permitirán, en el Ejercicio 3, reducir el tiempo necesario para montar una nueva máquina virtual que albergue el servidor MySQL, y así simplificar la migración de la aplicación a una configuración de dos tiers.

En el presente Ejercicio la máquina virtual sobre la que instalaremos Tomatocart usará como sistema operativo un Debian 7, que contendrá los paquetes y servidores exigidos como requerimientos por Tomatocart (Apache2, PHP5.2, y MySQL v5.0.7 o superior). Un requerimiento adicional será que la máquina virtual ocupe un volumen reducido de espacio de disco, por lo que no se instalarán en el Debian 7 los paquetes de interfaz gráfico de usuario, y se administrará usando la consola e texto.

Para realizar el ejercicio seguimos estos pasos:

1) Usar el procedimiento descrito en los puntos 1-2 del Ejercicio 2, Práctica 1, para crear en el directorio D: \practicas\MV\Debiantomato una máquina virtual llamada Debiantomato, con 8 GB de espacio de disco virtual thin.

2) A continuación, seguir el procedimiento descrito en los puntos 3 a 25 de ese mismo Ejercicio para hacer la instalación de un Debian 7, pero con los siguientes cambios:
  • El nombre de la máquina será debiantomato.
  • Se instalarán sólo los paquetes mostrados en la siguiente figura. (NOTA: Si la selección se ha hecho correctamente, el instalador nos indicará que se están descargando 143 paquetes).




Esta máquina virtual va a ofrecer funcionalidades vía un servidor web, y se conectará además a una base de datos (que inicialmente será local, pero que en el Ejercicio 3 pasará a ser remota). Es imprescindible, entonces, que tenga una IP estática, fija y conocida. Vamos a reconfigurar el servidor para ello:

3) Una vez reiniciado el servidor Debian, hacer login como root y editar el fichero /etc/network/interfaces para darle al servidor la dirección IP estática 10.0.0.21, como muestra la figura.



4) Editar el fichero /etc/hosts para añadir una línea con la resolución del nombre debiantomato a esa IP estática, como muestra la figura


5) Ejecutar update-rc.d exim4 disable (para evitar el timeout del servidor de correo durante el arranque).

6) Reiniciar la máquina, ejecutando reboot. Hacer de nuevo login como root.

Ahora vamos a instalar un conjunto de paquetes software adicionales (PHP5, MySQL, phpMyAdmin, unzip) que necesitaremos tanto para instalar Tomatocart como para administrar la base de datos MySQL asociada.

Dado que no hemos instalado soporte gráfico en el Debian, no podemos usar la herramienta synaptic. Aunque la gestión de los paquetes Debian se puede hacer, desde línea de comando, con la aplicación de bajo nivel apt-get, la existencia de diversas dependencias entre los paquetes (que deben ser correctamente resueltas) hace más aconsejable usar un interfaz administrativo de alto nivel para ello; usaremos entonces la herramienta aptitude, que tiene una funcionalidad similar a synaptic, pero que usa un interfaz de sólo-texto.

7) Para instalar los paquetesque nos faltan necesitamos que el DVD de Debian esté accesible en la lectora de la máquina virtual. Usando los procedimientos ya conocidos nos aseguramos de que la imagen ISO en el DVD virtual esté conectada, tras lo cual ejecutamos mount /media/cdrom; comprobamos el resultado ejecutando df.

8) Ejecutar aptitude. Usar Ctrl-T para abrir el menú e ir a la opción Buscar (o pulsar directamente la tecla /).



9) Buscar mysql. El listado de paquetes se desplazará hasta el primer nombre que contiene la cadena mysql. Seleccionar con el cursor el paquete mysql-server.



10) Pulsar + para añadir esta entrada a la lista de software pendiente de instalar. La entrada es un metapaquete; usando su información, aptitude determina que para conseguir esta funcionalidar deberá instalar el servidor mysql-server-5.5, y todos los paquetes de los que depende.



11) Buscar ahora php5. La primera entrada que aparece (libapache2-mod-php5) no nos sirve; pulsamos n para buscar la siguiente entrada, que ahora sí corresponde al metapaquete php5.



12) Seleccionar php5 y pulsar + para requerir su instalación; el sistema usa la información del metapaquete para seleccionar los paquetes que proporcionan el núcleo de PHP5. Adicionalmente, como se necesitan paquetes adicionales (en particular el soporte de acceso a MySQL) vamos a seleccionar (pulsando + sobre cada uno) el resto de los paquetes que comienzan por php5, como muestra la figura.



(NOTA: Nuestro servidor correrá en una red segura, por lo que esta selección no requiere mayores consideraciones. Sin embargo, en un servidor en producción y expuesto a Internet necesitaríamos ser más cuidadosos, e instalar sólo los paquetes estríctamente necesarios, para evitar dejar abierto accidentalmente algún agujero de seguridad a través de alguna vulnerabilidad explotable que pueda existir en esos paquetes).

13) Ahora instalaremos un paquete que proporciona un interfaz gráfico web de administración para MySQL. Para ello uamos de nuevo la función de búsqueda de aptitude, localizamos el paquete phpmyadmin y lo seleccionamos para instalar.



14) Por último, buscamos y seleccionamos para su instalación los paquetes unzip y zip (que proporcionan soporte para descomprimir ficheros .zip).

15) Ahora, debemos resolver manualmente conflictos de dependencias que aptitude no puede resolver por sí sólo. El mensaje en rojo de la parte inferior (que se puede observar en la figura anterior) nos avisa de la existencia de estos conflictos; pulsamos e para examinarlos.

16) La aplicación ofrece diversas soluciones para resolver los conflictos de dependencias; pulsando la tecla "." o la tecla "," se puede navegar por estas soluciones. Es necesario prestar atención a las acciones que propone la solución, porque algunas de ellas nos dejarán sin instalar los paquetes que necesitamos (acción [UNINST]). En nuestro caso, pulsaremos "." hasta llegar a la solución mostrada en la figura.



17) Pulsar la tecla "!" para aplicar esta solución, con lo que volvemos automáticamente a la ventana de selección de paquetes.

18) Ahora procedemos a instalar los paquetes seleccionados, pulsando la tecla "g". Aparecerá la ventana de previsualización de acciones, en la que, si queremos, podemos desplazarnos con el cursor por la lista de paquetes para comprobar que, efectivamente, se van a instalar los solicitados.



19) Pulsando nuevamente "g" comenzará la instalación. Durante la misma se nos pedirá la contraseña del usuario root del servidor de base de datos MySQL; usar deii15.

20) Durante la instalación se nos pedirá también elegir el servidor web que se debería configurar para que pueda ejecutarse phpMyAdmin; seleccionar apache2 (usando la barra espaciadora para marcar la selección).



21) Hacia el final de la instalación aparecerá un mensaje de configuración de phpMyAdmin. Aceptar la opción ofrecida por defecto.



22) A continuación se nos pedirá la contraseña para el usuario administrador de la base de datos phpMyAdmin; usar deii15. Usar también deii15 para la contraseña de aplicación MySQL (NOTA: sería probablemente más seguro dejarlo en blanco para que se genere una contraseña aleatoria).

23) Una vez terminada la instalación, pulsar q para salir de aptitude, y ejecutar reboot para reiniciar el servidor. Una vez reiniciado, hacer login como root.

24) Comprobamos que el servidor web funciona, abriendo en el Windows 7 una pestaña del Firefox y accediendo a http://10.0.0.21. Debemos obtener la respuesta mostrada en la figura.



25) Comprobamos que también funciona la herramienta administrativa phpMyAdmin, accediendo desde Firefox a http://10.0.0.21/phpmyadmin; deberemos obtener una respuesta como la de la figura.



Ahora, antes de continuar con la configuración de MySQL y la instalación de TomatoCart, vamos a crear una instantánea del estado actual del servidor Debian. Esta instantánea no sólo nos servirá para poder volver a ella caso de equivocarnos al instalar TomatoCart, sino que, en el Ejercicio 3, nos servirá para clonar rápidamente la máquina virtual que usaremos como servidor de base de datos en la arquitectura de 2 tiers.

26) Apagar el servidor, ejecutando telinit 0.

27) Usando el gestor de instantáneas, creamos una instantánea a la que llamaremos Sin Tomatocart.



28) Arrancar la máquina virtual, y hacer login como root.

El instalador del paquete Tomatocart es el fichero TomatoCart-1.1.8.6.1.zip, que está en el directorio D:\practicas\MV. Vamos a usar el cliente de SSH Bitwise Tunnelier para copiarlo al servidor Debian.

29) En el Windows 7, ejecutar Inicio -> Bitwise SSH Client -> Bitwise SSH Client.



30) Conectarse al servidor Debian, usando la configuración mostrada en la figura.



31) Aceptar la clave sólo para esta sesión, pulsando Accept for This Session.



32) Puede tardar unos cuantos segundos en pedir el password; esto es normal. Cuando aparezca la ventana de prompt, introducir el password de root (deii15).



(NOTA: La conexión directa como root es un agujero de seguridad obvio. En un sistema en producción debe configurarse el servidor SSH para que no permita directamente accesos remotos como root, y trabajar con una cuenta no privilegiada - como, por ejemplo, la del usuario curso que hemos creado durante la instalación).

33) Abrir una ventana de SFTP, pulsando Open New SFTP Window.



34) Transferir el fichero D:\practicas\MV\TomatoCart-1.1.8.6.1.zip al directorio /var/www (raíz de los documentos del servidor HTTP Apache2).

35) Una vez transferido, cerrar la ventana de SFTP y la xterminal abierta por el cliente SSH, y cerrar la aplicación Tunnerlier.

El usuario y grupo usados por el servidor apache son www-data (como puede verse haciendo more /etc/apache/envvars). Vamos a descomprimir el instalador y cambiar su usuario y grupo.

36) Hacer cd /var/www, y luego ejecutar unzip TomatoCart-1.1.8.6.1.zip

37) Cambiamos el nombre a la carpeta, haciendo mv TomatoCart-v1-released-v1.1.8.6.1 tomatocart 

38) Cambiamos el propietario y grupo de todos los ficheros del instalador, ejecutando chown -R www-data:www-data tomatocart 

39) Borramos del disco el fichero zip del instalador, haciendo rm TomatoCart-1.1.8.6.1.zip 

Ahora vamos a crear en el servidor MySQL la base de datos que necesita el instalador de TomatoCart. Para ello, usaremos el interfaz gráfico de administración phpMyAdmin.

40) En el Windows 7, abrir el navegador Firefox (o una nueva pestaña, si ya está abierto) y acceder a la URL http://10.0.0.21/phpmyadmin  .Cambiar el idioma a inglés, y hacer login como usuario root, como muestra la figura.



41) Pulsar en el botón Databases para acceder a la página de administración de las bases de datos existentnes en el servidor MySQL.



42) Crear una nuea base de datos, con el nombre tomatobase



43) Necesitamos crear un usuario administrador, distinto de root, para tomatobase. Accedemos para ello a la gestión de privilegios administrativos de esta base, pulsando check privileges



44) Pulsamos sobre Add a new User y rellenamos la información mostrada en la figura, con la que crearemos un usuario llamado tomatouser. Usar como password, de nuevo, deii15



45) Pulsamos el botón Go, al final del formulario. Obtendremos un mensaje (y una sentencia SQL), indicando que la creación del usuario se ha completado con éxito.



46) Hacemos logout de la herramienta phpMyAdmin, pulsando el botón al efecto, y cerramos esta pestaña en el navegador.



Ahora, instalaremos el software TomatoCart, usando para ello el instalador web que proporciona el mismo paquete.

47) Abrir una pestaña en el navegador Firefox, y acceder a la URL http://10.0.0.21/tomatocart/install

48) Marcar I agree to the license, y pulsar Continue



49) La comprobación previa a la instalación debe indicar que todos los settings y permisos son correctos. Pulsar Continue

50) Configuramos TomatoCart para que acceda a la base tomatobase del servidor local MySQL, empleando el usuario tomatouser. Para ello, rellenar el formulario con los valores mostrados en la figura, y pulsar Continue



51) Dejar con los valores por defecto la configuración del servidor web. Pulsar Continue



52) Vamos a dar valores iniciales a la tienda virtual, y a cargar los datos de ejemplo que proporciona el paquete para inicializar la tienda. En particular, en este formulario definimos un usuario administrador para la tienda virtual; usar el nombre tomatoadmin, y el password deii15, como se muestra en la figura. Pulsar Continue



La página final generada por el instalador nos proporcionará un mensaje confirmando que la instalación ha terminado correctamente, y dos botones que nos permiten ir, respectivamente, al front-end de la tienda (en la URL http://10.0.0.21/tomatocart) y al interfaz administrativo (http://10.0.0.21/tomatocart/admin).



53) Pulsar sobre el tobón Catalog para comprobar el funcionamiento del front-end de la tienda virtual.

Una vez que tenemos la tienda virtual funcionando vamos a configurarla, y a añadirle algunos artículos, usando su interfaz administrativo. Sin embargo, antes de empezar a cambiar cosas tomaremos otra instantánea del sistema, para poder volver atrás en caso de que los cambios que hagamos no sean satisfactorios.

54) Apagar el servidor debiantomato, ejecutando telinit 0

55) Usando el gestor de instantáneas crear una nueva instantánea, a la que llamaremos Con Tomatocart



56) Arrancar de nuevo la máquina virtual, y hacer login como root.

57) En el Windows 7 abrir Firefox, y acceder a la página de administración de Tomatocart en http://10.0.0.21/tomatocart/admin .Hacer login como usuario tomatoadmin

58) Al abrirse el interfaz administrativo arrancará un Wizard de configuración inicial de la tienda virtual. Rellenar los tres formularios con los valores mostrados en las figuras.


59) Vamos a necesitar poder visualizar qué efecto tienen en el front-end gráfico de la tienda virtual los cambios que realicemos a través del interfaz administrativo. Para ello, abrimos una segunda pestaña en Firefox, y accedemos en ella a la URL http://10.0.0.21/tomatocart

Examinando los datos de demo con que está inicializada nuestra tienda Deii, podemos ver que hay un error en la categoría Printers & Scanners. Si pulsamos sobre ella, comprobamos que los cuatro productos que aparecen corresponden, en realidad, a la categoría iPod & Camera. La primera modificación que haremos a través del interfaz administrativo será, entonces, corregir este error.

60) En el interfaz de administración de TomatoCart pulsar sobre Products (o, alternativamente, acceder a través del botón de inicio de la parte inferior, a través de Start -> Catalog -> Products -> Products).


61) En la ventana de Productos, si pulsamos sobre la categoría Printers & Scanners podemos comprobar que, en efecto, en ella aparecen productos que deberían estar bajo iPod & Camera



62) Primero, moveremos la cámara y los iPods a la categoría correcta. Para ello, seleccionamos con el ratón los cuatro productos (usando Shift para permitir selecciones múltiples), y los arrastramos sobre la carpeta iPod & Camera del árbol de categorías. Una vez arrastrados, pulsamos sobre iPod & Camera para comprobar que los productos aparecen ahora en esta categoría.



63) Como todavía no hemos publicado los cambios, el front-end de la tienda todavía no ha sido modificado (como podemos verificar volviendo a la pestaña de la tienda Deii, refrescando con F5, y comprobando que siguen apareciendo 0 artículos bajo iPod & Camera).

64) Volvemos al interfaz administrativo, seleccionamos los cuatro productos de la categoría iPod & Camera, y pulsamos el botón Publish. Volvemos entonces a la tienda virtual, refrescamos con F5, y comprobamos que los productos están ahora en la categoría correcta.

Obviamente sigue existiendo un problema, porque los iPods y cámara siguen apareciendo también bajo la categoría de Printers & Scanners. Adicionalmente, deberíamos separar los iPods de la cámara, usando subcategorías. Vamos a realizar ambas modificaciones.

65) En la ventana Products del interfaz administrativo pulsar sobre Printers & Scanners. Seleccionar los cuatro productos de esta categoría, y pulsar Delete. Esta acción provoca una publicación automática, como podemos comprobar refrescando la tienda virtual y viendo que ahora hay 0 productos bajo Printers & Scanners..... Oooooppsss......!!!!!

...Al comprobar el front-end de la tienda, nos damos cuenta de que este borrado ha sido un error. Como podemos ver, ¡¡ahora también hay 0 artículos bajo iPod & Camera!!. Obviamente el arrastre que hicimos en el punto 62 no había hecho una copia de los productos, sino sólamente creado un enlace, a nivel de la base de datos, desde esa categoría a los productos ya existentes, de forma que el borrado de los productos los ha eliminado simultáneamente de ambas categorías.

Justamente para resolver este tipo de problemas es para lo que hemos tomado la instantánea Con Tomatocart, que usaremos ahora para recuperar el estado inicial de la tienda virtual.

66) Apagar el servidor debiantomato usando telinit 0 y, en el menú de instantáneas, revertir a la instantánea Con Tomatocart. Volver a encender el servidor, y hacer login como root.

67) Accediendo desde Firefox a la tienda virtual comprobamos que volvemos a tener su estado inicial. Sin embargo, al hacer login en el interfaz administrativo de la tienda vemos que también deberemos volver a rellenar los formularios del Wizard. Hacerlo con los mismos valores que ya usamos anteriormente (en el punto 58).

Es evidente que cada vez que volvamos atrás a la instantánea Con Tomatocart tendremos que volver a rellenar los formularios del Wizard. Sería cómodo que, en esta fase en que estamos configurando la tienda por "tanteo y error", pudiéramos volver atrás rápidamente a un estado en el que el Wizard ya estuviera relleno y la tienda inicilizada, pero resulta engorroso tener que parar la máquina completa para hacer una instantánea que, probablemente, tendrá una vida útil muy limitada. Por ello, vamos a usar ahora una característica de las instantáneas que nos soluciona esta necesidad: la posibilidad de crear una instantánea con la máquina encendida. Esta instantánea no sólo congela el estado actual del disco duro, sino que también guarda el contenido actual de la memoria RAM, de forma que al recuperarla también recuperamos el estado que tenían los procesos en el instante en que se creó esa instantánea.

68) Sin apagar ni suspender debiantomato abrir en el VMware Workstation el gestor de instantáneas, y crear una instantánea llamada Tras Wizard. Observar las flechas verdes, que indican máquina en funcionamiento.



69) Para probar el funcionamiento de esta instantánea, en el interfaz administrativo de TomatoCart borrar los cuatro productos bajo Printers & Scanners. Comprobar en el front-end de la tienda que ahora hay 0 productos en esa categoría.

70) Volver a la ventana del VMware Workstation y ejecutar VM -> Snapshot -> Revert to Snapshot: Tras Wizard. La máquina se parará y volverá a arrancar (como si saliese de una suspensión).

71) Refrescar el interfaz administrativo de TomatoCart, y el front-end de la tienda, y comprobar que hemos vuelto a la situación anterior al borrado (es decir, conectados al interfaz administrativo, con el Wizard ya relleno, y con los cuatro productos originales bajo Printers & Scanners).

Ahora vamos a cambiar correctamente la filiación de categoría para estos cuatro productos. Adicionalmente, aprovecharemos para separarlos en dos subcategorías, iPod y Camera.

72) Seleccionar la cámara y pulsar su botón Edit.



73) En la pestaña de Categoríes, activar su filiación a iPod & Camera, y desactivar la de HP. Pulsar Submit.



74) Refrescar el front-end de la tienda, para comprobar el resultado.

75) Antes de mover los iPods crearemos las subcategorías bajo iPod & Camera. Usando el botón derecho, abrir el menú contextual Add, y añdir la subcategoría iPod, como muestran las figuras.

76) Repetir el procedimiento para crear la subcategoría Camera. Refrescar el front-end de la tienda para ver el resultado.



77) Usando el procedimiento descrito en los puntos 72 y 73, mover los tres iPods a la subcategoría iPod, y la cámara Sony a la subcategoría Camera. Comprobar el resultado en el front-end de la tienda.

Ahora añadiremos a la tienda virtual un producto nuevo: una impresora HP Laserjet P1102w.

78) En la ventana Products del interfaz administrativo de TomatoCart seleccionar la categoría Printers & Scanners -> HP, y pulsar el botón Add.



79) Rellenar las pestañas General, Meta Info, Data y Categories con los valores que muestran las siguientes imágenes; pulsar Save and Continue Edit en todos los casos (NOTA: la descripción de la impresora se puede copiar directamente de la página de este producto en el web de HP).

80) Ahora cargaremos en el servidor debiantomato las imágenes del producto, que tenemos en el Windows 7. En la pestaña Images pulsar el botón Add bajo Remote File Upload. Se abrirá un navegador de ficheros; buscar el directorio D:\practicas\MV\P1102 y seleccionar la imagen c01953212_390x286.jpg. Pulsar Upload; la imagen es transferida al servidor, y asignada al producto. Repetir con el resto de las imágenes del directorio P1102, y pulsar Submit.

81) Actualizar el front-end de la tienda, y comprobar cómo la impresora aparece en Printers & Scanners, bajo la subcategoría HP.

Una vez que hemos conseguido una configuración satisfactoria del front-end de la tienda, vamos a eliminar todas las instantáneas intermedias (excepto la que nos servirá luego para clonar el servidor de base de datos), y crearemos una instantánea que nos servirá de punto de partida para el siguiente Ejercicio.

82) Apagar, usando telinit 0, el servidor debiantomato, y abrir el gestor de instantáneas.

83) Seleccionar simultáneamente las instantáneas Con Tomatocart, Tras Wizard y Snapshot 4, y borrarlas.

84) Crear una nueva instantánea llamada Tomatocart final, y cerrar el gestor de instantáneas.



Con este paso terminamos el Ejercicio.
